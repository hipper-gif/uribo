# Uribo 事実台帳（facts）

> **役割**: 相手（客・利用者・現場）が教えてくれた「相手側の環境・運用・体制・数字の事実」を1行1件で記録する検索用台帳。**相手側の現実だけ**を書く（自分の約束・提案・決定は書かない → 約束は `docs/promises.md`、決定は `docs/decisions/`）。
> **なぜ**: 要約・記憶で客に返すと「前に聞いた事実」を取りこぼす。ここを全読みして既出チェックする（lossy-context対策）。参照実装＝EnrollHub `docs/facts.md`。

## 運用ルール（4条）

1. **返信・設計判断の前に、この台帳と `docs/transcripts/` を検索する（既出チェック）。** 返信案の質問文は全件チェックし、ヒットしたら「〜と伺っています。変わっていれば教えてください」の**確認形**に変換する（確認形は既出でも信用を落とさない）。
2. **相手からの受信・打ち合わせのたびに事実を追記する**（1行1文＋出典）。
3. **事実が変わったら行を書き換え、出典に変更日を追記**する（古い事実を残さない）。
4. これは**相手側の現実**の台帳。自分側の約束・提案・決定は書かない。
